Versionamento de código


Trata-se um sistema que gerencia e rastreia alterações no código-fonte ao longo do tempo. Ele facilita a colaboração entre desenvolvedores e permite reverter a versões anteriores do código. Ferramentas como Git e GitHub são amplamente utilizadas para este propósito.

GIT

GIT é um sistema de controle de versão distribuído, desenvolvido por Linus Torvalds em 2005. Ele permite que desenvolvedores rastreiem mudanças no código-fonte ao longo do tempo, colaborem de forma eficiente e revertam a versões anteriores se necessário. Cada cópia do repositório Git é independente, garantindo segurança e redundância.

Não confunda com GITHUB! Este é uma plataforma de hospedagem de código-fonte baseada na web que usa o Git para controle de versão. Além de armazenar repositórios Git, GitHub oferece uma interface visual, ferramentas para colaboração em equipe, integração contínua, gerenciamento de projetos, e funcionalidades de revisão de código. Ele facilita a colaboração em projetos de software, tanto públicos quanto privados. É a plataforma mais famosa porém não a única.

Instalando e configurando o GIT no seu computador

Windows: baixe o instalador do Git no site oficial Git for Windows e siga as instruções de instalação.

MacOS: use o Homebrew com o comando brew install git. Linux: use o gerenciador de pacotes da sua distribuição (por exemplo, sudo apt-get install git para distribuições baseadas no Debian).

Para configurar seu nome e usuário, clique do lado direito do mouse e opção "Git Bash Here" para abrir o terminal. Digite os comandos abaixo para configurar.


 git config --global user.name "Seu Nome"
 git config --global user.email "seu.email@example.com"

Chaves SSH

O que são?

As chaves SSH são um par de chaves criptográficas (uma pública e uma privada) usadas para autenticar você em servidores remotos, como o GitHub. Elas proporcionam uma maneira segura e conveniente de autenticação sem a necessidade de inserir sua senha constantemente.

Criando no seu computador

Abrindo o terminal com Git Bash Here digite o comando ssh-keygen -t ed25519 -C "your_email@example.com". Verifique esta documentação caso não consiga com o código acima.

Siga as instruções, isso irá gerar duas chaves: uma pública e outra privada.

Adicionando sua chave pública ao GitHub

Vá para o perfil, configurações, e então clique em SSH e GPG Keys.

Copie sua chave pública que está salva no seu computador (dentro do perfil, na pasta SSH) e cole o conteúdo no campo "Key" e dê um nome descritivo (identificação do computador). Pronto! Agora você pode clonar, puxar e enviar alterações para seus repositórios GitHub usando autenticação SSH.
