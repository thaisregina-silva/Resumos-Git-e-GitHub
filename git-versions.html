Gerenciando versões com GIT

Salvando as primeiras versões do projeto usando o terminal

Um projeto é criado no seu computador com seus arquivos e documentos. Para iniciar o versionamento de código usando o GIT no seu computador dê um "Git Bash Here" na pasta do projeto e insira os comandos a seguir no terminal: 


git init     //inicia criação de repositórios

git add.     //prepara os arquivos na área stage - aqui ele irá selecionar todos//

git commit -m "Mensagem Explicativa da Versão"

git branch -M main     //para garantir que será salvo como Main - e não master//

git remote add origin git@github.com:seuusuario/seurepositorio.git     //linka o projeto local com o repositório GitHub

git push -u origin main     //envia o projeto para o GitHub

Quando preparamos o projeto para enviar usando o comando git add ., podemos conferir tudo que está preparado para envio com o comando git status. Sempre que precisar checar o status do projeto com GIT (quais arquivos serão enviados, se há modificações não salvas etc.), use esse comando.

Salvando as primeiras versões usando o GitHub Desktop

A próxima página traz mais detalhes sobre instalação e configuração da ferramenta, mas o uso dela para salvar as versões é bastante prático e intuitivo.

Para cada edição que necessite de commit, na tela principal do GitHub Desktop (observado se está na branch correta) há um campo para inserir a mensagem do commit e seus comentários. Para salvar a versão clique em commit e para enviar para repositório remoto dê um push. Sempre que quiser checar se o repositório remoto está atualizado com o local, dê um fetch.
