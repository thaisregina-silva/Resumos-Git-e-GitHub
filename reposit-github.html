Repositórios no GitHub

Definição

São os locais de armazenamento online onde ficam salvos os projetos de código. Cada repositório contém todos os arquivos do projeto e o histórico de alterações. Repositórios podem ser públicos, acessíveis por qualquer pessoa, ou privados, acessíveis apenas por usuários autorizados. Eles são salvos nos servidores do GitHub, garantindo segurança e acessibilidade.

Criando os seus repositórios

Na página inicial do GitHub, clique em NEW para criar um novo repositório. Defina um nome e descrição. Quando criado, o próprio GitHub irá trazer a linha de código para inserir no terminal após salvar e preparar os arquivos do projeto para enviar ao GitHub:git remote add origin git@github.com: seuusuario/seurepositorio.git.

Certifique-se que está usando a URL corretamente.Escolha SSH e não HTTPS. Para corrigir a URL use o comando git remote set-url origin git@github.com/seuusuario/seurepositorio.git

Após criado, o repositório irá trazer as páginas e arquivos criados e anexados, número de branchs e commits (e seus criadores), além do arquivo README.

Arquivo README

O que é?

O arquivo README é um documento de texto, geralmente escrito em formato Markdown (.md), presente na raiz de um repositório. Ele fornece informações essenciais sobre o projeto, como sua finalidade, como configurá-lo, usá-lo e contribuir com ele.

Por que usá-lo?

Criar um README para seu projeto é importante pois ele facilita a compreensão fornecendo uma visão geral clara do projeto, seu propósito e funcionalidades. Além disso ele comunica padrões (boas práticas, convenções e guias de contribuição) e serve como guia para colaboradores ajudando a entenderem como configurar o ambiente e começar a trabalhar no projeto. Ainda, um README bem escrito pode tornar o projeto mais atrativo e acessível para outros desenvolvedores.

Dicas para estruturar seu README

Título: Comece com um título claro que represente o nome do projeto.
Descrição: Inclua uma breve descrição do projeto e seus objetivos.
Instalação: Forneça instruções detalhadas sobre como instalar e configurar o projeto.
Uso: Adicione exemplos de uso e comandos básicos.
Contribuição: Informe como outros desenvolvedores podem contribuir com o projeto.
Licença: Inclua informações sobre a licença do projeto.
Contato: Adicione formas de contato para dúvidas ou suporte.

Para facilitar, podemos usar o site https://readme.so/pt para redigir o arquivo README.

Evite: descrições vagas ou genéricas, documentação desatualizada, excesso de jargão (evite termos técnicos complexos sem explicação) e informações irrelevantes ao projeto.
