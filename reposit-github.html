Repositórios no GitHub

São os locais de armazenamento online onde ficam salvos os projetos de código. Cada repositório contém todos os arquivos do projeto e o histórico de alterações. Repositórios podem ser públicos, acessíveis por qualquer pessoa, ou privados, acessíveis apenas por usuários autorizados. Eles são salvos nos servidores do GitHub, garantindo segurança e acessibilidade.

GitHub Desktop

O GitHub Desktop é uma aplicação que facilita a interação com repositórios Git por meio de uma interface gráfica amigável. Projetado para simplificar o fluxo de trabalho, ele permite que desenvolvedores façam commit, push, pull, merge e resolvam conflitos de código sem precisar utilizar a linha de comando. A ferramenta é especialmente útil para iniciantes que estão começando a utilizar o Git, bem como para desenvolvedores experientes que buscam maior eficiência e uma visualização clara das alterações de código. Você pode baixar o GitHub Desktop acessando este link.

Criando seus repositórios via GitHub Desktop

O aplicativo é simples de usar. A barra de ferramentas oferece todas as opções, dentre criar repositório, deletar, criar branches, entre outras.

Dentro de File -> New Respository você cria um novo repositório. Insira as informações do seu repositório. Na tela irão aparecer opções como Push (enviar para remoto) e Open with VSCode (para iniciar a edição do código).

Criando os seus repositórios via terminal

Na página inicial do GitHub, clique em NEW para criar um novo repositório. Defina um nome e descrição. Quando criado, o próprio GitHub irá trazer a linha de código para inserir no terminal após salvar e preparar os arquivos do projeto para enviar ao GitHub:git remote add origin git@github.com: seuusuario/seurepositorio.git.

Certifique-se que está usando a URL corretamente.Escolha SSH e não HTTPS. Para corrigir a URL use o comando git remote set-url origin git@github.com/seuusuario/seurepositorio.git

Após criado, o repositório irá trazer as páginas e arquivos criados e anexados, número de branchs e commits (e seus criadores), além do arquivo README.

Arquivo README

O arquivo README é um documento de texto, geralmente escrito em formato Markdown (.md), presente na raiz de um repositório. Ele fornece informações essenciais sobre o projeto, como sua finalidade, como configurá-lo, usá-lo e contribuir com ele.

Criar um README para seu projeto é importante pois ele facilita a compreensão fornecendo uma visão geral clara do projeto, seu propósito e funcionalidades. Além disso ele comunica padrões (boas práticas, convenções e guias de contribuição) e serve como guia para colaboradores ajudando a entenderem como configurar o ambiente e começar a trabalhar no projeto. Ainda, um README bem escrito pode tornar o projeto mais atrativo e acessível para outros desenvolvedores.

Dicas para estruturar seu README

Título: Comece com um título claro que represente o nome do projeto.
Descrição: Inclua uma breve descrição do projeto e seus objetivos.
Instalação: Forneça instruções detalhadas sobre como instalar e configurar o projeto.
Uso: Adicione exemplos de uso e comandos básicos.
Contribuição: Informe como outros desenvolvedores podem contribuir com o projeto.
Licença: Inclua informações sobre a licença do projeto.
Contato: Adicione formas de contato para dúvidas ou suporte.

Para facilitar, podemos usar o site https://readme.so/pt para redigir o arquivo README.

Evite: descrições vagas ou genéricas, documentação desatualizada, excesso de jargão (evite termos técnicos complexos sem explicação) e informações irrelevantes ao projeto.

Clonando repositórios

Clonar um repositório significa criar uma cópia exata dele no seu computador, incluindo todo o histórico de commits. Isso é útil quando você quer trabalhar em um projeto existente sem precisar criá-lo do zero. No GitHub Desktop, clique em "File" > "Clone repository", escolha o repositório e selecione onde salvá-lo. No terminal (Git Bash Here), use:

git clone URL_DO_REPOSITORIO

Cuide para não modificar diretamente o main sem permissão. Para contribuir, crie uma branch separada.

Fazendo forks

Um fork é uma cópia independente de um repositório para sua conta, permitindo modificar o código sem afetar o original. Isso é útil para contribuir com projetos open-source. No GitHub, clique em "Fork" na página do repositório. Depois, clone seu fork localmente para trabalhar nele. No terminal:

git clone https://github.com/seu-usuario/repo-fork.git

Quando terminar as mudanças, envie um Pull Request (PR) para sugerir a atualização no repositório original. Dica: Sempre sincronize seu fork com o repositório original antes de começar novas mudanças.

Issues

As issues são usadas para relatar bugs, sugerir melhorias ou discutir algo sobre o projeto. Você pode criá-las na aba "Issues" do repositório no GitHub, clicando em "New Issue". Seja claro no título e descrição, anexando prints ou trechos de código quando necessário.

Importante: se for contribuir com a solução de uma issue, procure se há há um tópico sobre e comente na própria issue antes de começar, para evitar trabalho duplicado.
